# System Design - From Scratch to Expert

A comprehensive, **first-principles system design curriculum** that takes you from computing fundamentals all the way to advanced distributed systems and frontend system design. Every concept is taught deeply - _what it is, why it exists, how it works internally, and its trade-offs_ - and grounded in **real-world engineering from top companies** (Meta, Netflix, AWS, Google, Stripe, and more).

Learning-first, not interview cram - but it covers everything a **system design interview** at a top tech company expects.

## What's inside

**Backend / Distributed-Systems track (F, L0–L15)**
Computing fundamentals (CPU, memory, concurrency, I/O, serialization, hashing) · networking (DNS, TCP/HTTP, TLS, load balancers, CDN) · relational & NoSQL databases · caching · message queues & streaming (Kafka) · distributed systems theory (CAP, consensus/Raft, consistency, CRDTs) · reliability & SRE · observability · security · API & service design · architecture patterns (microservices, event-driven, cell-based) · scalability & performance patterns · specialized systems (search, geospatial, recommendation, ML/LLM serving) · cloud & infrastructure (Kubernetes, IaC) · applied design problems.

**Frontend System-Design track (FE-F, FE-L0–FE-L8)**
Browser internals & rendering (CSR/SSR/SSG/RSC) · client data & state · Core Web Vitals & performance · real-time & offline (PWA, CRDT) · tooling & testing · accessibility, i18n & design systems · frontend observability & security · FE design problems.

## How it's organized

- **`roadmap.md`** - the full syllabus with per-topic status and priority.
- **`lessons/`** - bite-sized, digestible lessons (one concept per file, with diagrams and worked examples).
- **`research/`** - dense concept references (the source of truth) plus verified sources.
- **`practice/`** - design problems, attempts, and reviews.
